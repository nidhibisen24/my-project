जावा एक्सेप्शन क्या हैं? (Java Exceptions in Hindi)

यदि हम शब्दकोष में exception का अर्थ देखें तो इसका अर्थ है abnormal condition। जावा में, exception एक unexpected or unwanted event है जो प्रोग्राम Execution, यानी रनटाइम के दौरान होती है और प्रोग्राम के नियमित Flow को बाधित करती है।

यह प्रोग्राम या एप्लिकेशन को असामान्य रूप से residual करता है, जो एक ideal situation नहीं है। इसलिए, ऐसे exception को skillfully संभालना आवश्यक है।

हम प्रोग्राम द्वारा जावा में exceptions को संभाल और पकड़ सकते हैं। जब कोई exception होता है, तो यह एक ऑब्जेक्ट बनाता है जिसे exception ऑब्जेक्ट के रूप में जाना जाता है। इसमें जावा में exception के बारे में महत्वपूर्ण जानकारी शामिल है, जिसमें exception का Description and name, program की स्थिति आदि शामिल है।

जावा में एक्सेप्शन क्यों होते हैं? (Why Do Exceptions Happen in Java in Hindi?)

जावा exception विभिन्न कारणों से हो सकता है। यहां कुछ scenarios दिए गए हैं जब exception हो सकता है:

  • जब कोई यूजर कोई invalid इनपुट या डेटा दर्ज करता है

  • जब communication के दौरान नेटवर्क कनेक्शन lost हो जाता है

  • जेवीएम की मेमोरी खत्म हो गई है

  • जब कोई आवश्यक फ़ाइल नहीं मिल पाती या अनुपलब्ध होती है

  • Code errors

  • Device failure

Exception हैंडलिंग के लाभ हैं:-

  • program execution को पूरा करने का प्रावधान

  • error types की आसान पहचान

  • Error Management और प्रोग्राम कोड की आसान पहचान

  • efficient error reporting

  • Propagation of errors

Java Exception Hierarchy in Hindi

जावा में exceptionों को एक hierarchical structure में व्यवस्थित किया जाता है जिसे जावा exception hierarchy के रूप में जाना जाता है। यह hierarchical inheritance की concept पर आधारित है, जहां माता-पिता-बच्चे के रिश्ते में exceptions को व्यवस्थित किया जाता है।

 

Hierarchy के शीर्ष पर throwable class है, जो जावा में सभी exceptions के लिए आधार class के रूप में कार्य करता है।

थ्रोएबल क्लास के दो मुख्य subclasses हैं:-

1. Error: 

Represents serious का Representation करता है जो आम तौर पर प्रोग्राम के Control से परे होती हैं, जैसे system failures या resource exhaustion। उदाहरणों में आउटऑफमेमरी एरर और स्टैक ओवरफ्लो एरर शामिल हैं। errors आमतौर पर applications द्वारा caught or control नहीं की जाती हैं।

2. Exception: 

Represents exceptional का प्रतिनिधित्व करता है जिन्हें applications द्वारा caught or control किया जा सकता है।

 Exception class आगे दो subclasses में divided होता है:-

  • RuntimeException: उन exception का प्रतिनिधित्व करता है जो रनटाइम पर होते हैं और आमतौर पर प्रोग्रामिंग errors या invalid operation के कारण होते हैं। उदाहरणों में NullPointerException और ArrayIndexOutOfBoundsException शामिल हैं। ये exception unchecked हैं, जिसका अर्थ है कि उन्हें legal signature में स्पष्ट रूप से घोषित करने या कॉलिंग कोड द्वारा पकड़े जाने की आवश्यकता नहीं है।

  • Checked exceptions: उन exception का प्रतिनिधित्व करते हैं जिन्हें method signature में स्पष्ट रूप से घोषित किया जाना चाहिए या कॉलिंग कोड द्वारा पकड़ा जाना चाहिए। ये exception, exception के subclasses हैं लेकिन RuntimeException के subclasses नहीं हैं। उदाहरणों में IOException और SQLException शामिल हैं। चेक किए गए exception का उद्देश्य डेवलपर द्वारा उचित Error Management को लागू करना है।

Exception class में कई subclasses हैं जो various error conditions को संभालने के लिए अधिक विशिष्ट प्रकार के Exception प्रदान करते हैं।

कुछ notable subclasses में शामिल हैं:-

  • IOException: इनपुट/आउटपुट Operation से संबंधित exceptionों का प्रतिनिधित्व करता है, जैसे फ़ाइल हैंडलिंग या नेटवर्क संचार।

  • NullPointerException: इंगित करता है कि एक void reference तक पहुंच प्राप्त की गई थी।

  • ClassNotFoundException: रनटाइम पर क्लास नहीं मिलने पर थ्रो कर दिया जाता है.

  • ArrayIndexOutOfBoundsException: एक invalid array index पहुंच को point out करता है।

  • NumberFormatException: इंगित करता है कि एक स्ट्रिंग को numeric format में पार्स नहीं किया जा सकता है।

Application-specific error scenarios को संभालने के लिए डेवलपर्स exception class या उसके subcategories का विस्तार करके custom exception classes भी बना सकते हैं।

जावा में एक्सेप्शन के प्रकार (Types of Exceptions in Java in Hindi)

जावा में मुख्य रूप से दो प्रकार के exception हैं, प्रत्येक इसके different class libraries से संबंधित हैं। साथ ही, उपयोगकर्ता अपने स्वयं के exception को परिभाषित कर सकते हैं। आइए बेहतर समझ के लिए जावा exception प्रकारों के बारे में गहराई से जानें:-

  1. User-defined

  2. Built-in

1. User-defined Exceptions in Java

जावा आपको अपने स्वयं के Exception बनाने की अनुमति देता है, क्योंकि उपलब्ध exception कुछ स्थितियों को परिभाषित नहीं कर सकते हैं। इन exception को user-defined exception के रूप में जाना जाता है।

हालाँकि, जब आप अपने exception लिखते हैं, तो आपको निम्नलिखित बिंदुओं पर ध्यान देने की आवश्यकता है:-

  • सभी exception थ्रोएबल से प्राप्त होने चाहिए।

  • रनटाइम exception लिखने के लिए, आपको रनटाइम क्लास का विस्तार करना होगा।

  • चेक किए गए exception को लिखने के लिए आपको exception Class का विस्तार करना होगा जिसे हैंडल या डिक्लेयर नियम द्वारा स्वचालित रूप से लागू किया जा सकता है। यह फ़ील्ड और Methods वाले किसी भी अन्य Class की तरह है।

Syntax for User-defined Java Exception

अपने स्वयं के exception को परिभाषित करने या उपयोगकर्ता द्वारा परिभाषित exception बनाने के लिए, यहां syntax है:-

class MyException extends Exception {
}

उपयोगकर्ता-परिभाषित exception बनाने के लिए आपको बस पूर्वनिर्धारित exception Class का विस्तार करने की आवश्यकता है। इसे चेक किए गए exception के रूप में जाना जाता है।

2. Built-in Exceptions in Java

Built-in exceptions जावा libraries में उपलब्ध हैं और विभिन्न error conditions का वर्णन करने के लिए उपयुक्त विकल्प हैं।

जावा में एक implicit exception हो सकता है:-

a) Checked Exceptions

जाँचे गए exception को compile-time exception के रूप में भी जाना जाता है क्योंकि उन्हें compile time पर compiler द्वारा जाँचा जाता है। प्रोग्रामर इन exceptions को अनदेखा नहीं कर सकता, बल्कि उसे जावा में इन exceptions को संभालना होगा।

जावा में जाँचे गए exception के उदाहरण:-

  • SQLException 

  • IOException

b) Unchecked Exceptions

ये exception चेक किए गए exception के विपरीत हैं। वे Execution के समय होते हैं और उन्हें रनटाइम exception के रूप में भी जाना जाता है। कंपाइलर कंपाइल समय पर unchecked exceptions की जांच नहीं करता है।

यदि कोई प्रोग्रामर किसी unchecked java exception को handling or declaring किए बिना Throw कर देता है, तो प्रोग्राम एक compilation error देता है।

जावा में unchecked exceptions की सूची में निम्नलिखित उदाहरण शामिल हैं:-

  • ArrayIndexOutOfBoundsException

  • NullPointerException

  • ArithmeticException 

Logic errors, प्रोग्रामिंग बग और एपीआई का अनुचित उपयोग भी unchecked exceptions के उदाहरण हैं।

जावा में Error क्या हैं? (Errors in Java in Hindi)

जावा में Errors and Exceptions पूरी तरह से अलग concepts हैं। errors users और प्रोग्रामर के नियंत्रण से परे की समस्याएँ हैं। इसलिए, हम उन्हें बिल्कुल भी संभाल नहीं सकते। कोड लिखते समय उन्हें नजरअंदाज कर दिया जाता है क्योंकि उनके बारे में हम शायद ही कुछ कर सकते हैं। इसके अलावा, Compilation के समय errors को भी नजरअंदाज कर दिया जाता है।

जावा में errors के उदाहरणों में irreversible conditions शामिल हैं, जैसे Library incompatibility, memory leak, infinite recursion, Java Virtual Machine की मेमोरी खत्म हो जाना, स्टैक ओवरफ़्लो, आदि।

किसी भी Confusion से बचने के लिए, errors and exceptions के बीच अंतर यहां दिया गया है:-

Errors एक Serious Problem का प्रतिनिधित्व करती हैं जो हमारे Control से परे है, इसलिए एक उचित एप्लिकेशन को इसे खोजने का प्रयास नहीं करना चाहिए।

Exception उन स्थितियों को referred करते हैं जिन्हें हम संभाल सकते हैं और जिन्हें एक proper application पकड़ सकता है।

जावा में एक्सेप्शन इनफार्मेशन प्रिंट कैसे करें?

Debugging and error handling में सहायता के लिए आप exception जानकारी को कंसोल या लॉग फ़ाइलों पर प्रिंट कर सकते हैं।

जावा में exception जानकारी प्रिंट करने के कई तरीके हैं:-

Using printStackTrace(): 

थ्रोएबल क्लास की प्रिंटस्टैकट्रेस () Method exception के स्टैक ट्रेस को प्रिंट करती है, जिसमें exception तक पहुंचने वाली Method कॉल का sequence शामिल होता है। इसका उपयोग अक्सर quick debugging और exception के कारण को समझने के लिए किया जाता है। यहाँ एक उदाहरण है:

try {
    // Code that may throw an exception
} catch (Exception e) {
    e.printStackTrace(); // Prints the stack trace to the console
}

Using getMessage(): 

GetMessage() Method exception का Brief summary लौटाती है। इसका उपयोग exception से जुड़े एक specific message को प्राप्त करने के लिए किया जा सकता है। यहाँ एक उदाहरण है:-

try {
    // Code that may throw an exception
} catch (Exception e) {
    System.out.println(e.getMessage()); // Prints the exception message to the console
}

Using a logging framework: 

अधिक complex applications में, अक्सर Log4j, SLF4J, या Java की built-in logging (java.util.logging) जैसे लॉगिंग फ्रेमवर्क का उपयोग करने की अनुशंसा की जाती है। ये फ्रेमवर्क अधिक Advanced logging capabilities प्रदान करते हैं, जिसमें विभिन्न लॉग स्तर, कॉन्फ़िगर करने योग्य output destination और additional exceptions जानकारी शामिल है।

यहां जावा की built-in logging का उपयोग करके एक उदाहरण दिया गया है:-

import java.util.logging.Logger;
class MyClass {
    private static final Logger LOGGER = Logger.getLogger(MyClass.class.getName());
    void myMethod() {
        try {
           // Code that may throw an exception
        } catch (Exception e) {
            LOGGER.severe("An exception occurred: " + e.getMessage());
            LOGGER.severe("Stack trace:");
            for (StackTraceElement element : e.getStackTrace()) {
                LOGGER.severe(element.toString());
          }
      }
    }
}

यहां, exception जानकारी java.util.logging से लॉगर क्लास का उपयोग करके लॉग की गई है। लॉग स्तर severe पर सेट है, लेकिन आप अपनी आवश्यकताओं के आधार पर उचित लॉग स्तर चुन सकते हैं।

exception जानकारी प्रिंट करके, आप अपने जावा applications में समस्याओं के Diagnosis and Solution में सहायता के लिए exception के कारण, Method कॉल के sequence और अन्य relevant details में insight प्राप्त कर सकते हैं।

How Does JVM Handle Exceptions in Hindi

किसी Method में, जब कोई exception होता है, तो यह Method को exception ऑब्जेक्ट नामक ऑब्जेक्ट बनाने की ओर ले जाता है, और यह ऑब्जेक्ट को रनटाइम सिस्टम, यानी, जेवीएम में भेज देता है। इस ऑब्जेक्ट में exception वाले प्रोग्राम की present situation के साथ exception का नाम और Description शामिल है।

एक exception ऑब्जेक्ट बनाने और उसे रनटाइम में संभालने की इस प्रक्रिया को exception throw कहा जाता है। Methods की एक sorted list, जिसे कॉल स्टैक के रूप में जाना जाता है, उस Method तक पहुंचने के लिए कॉल की जाती है जहां exception हुआ है।

इसके बाद दी गई प्रक्रिया अपनाई जाएगी:

रनटाइम सिस्टम कोड के ब्लॉक के साथ मेथड को देखने के लिए कॉल स्टैक की खोज करता है, जिसे exception हैंडलर के रूप में जाना जाता है, जो exception को संभाल सकता है।

रनटाइम सिस्टम उस मेथड की खोज करता है जहां exception हुआ और कॉल स्टैक के माध्यम से उल्टे क्रम में चलता है जिसमें यह Methods को कॉल करता है।

यदि उसे सही और उपयुक्त हैंडलर मिल जाता है, तो वह उसे exception भेज देता है।

कॉल स्टैक पर सभी तरीकों को खोजने के बाद भी, यदि रनटाइम को उपयुक्त हैंडलर नहीं मिल पाता है, तो यह exception ऑब्जेक्ट को डिफ़ॉल्ट exception हैंडलर में भेज देता है। हैंडलर दिए गए Format में exception जानकारी प्रिंट करता है और प्रोग्राम को अचानक बंद कर देता है।

Java Exception Keywords in Hindi

1. try: 

ट्राई ब्लॉक का उपयोग उस कोड को attached करने के लिए किया जाता है जो exception थ्रो कर सकता है। इसके बाद एक या अधिक कैच ब्लॉक या finally block (or both) आते हैं।

2. catch: 

कैच ब्लॉक का उपयोग exception को catch and handle के लिए किया जाता है। यह ट्राई ब्लॉक का Pursuance करता है और उस exception के प्रकार को Specified करता है जिसे वह संभाल सकता है। जब उस प्रकार का exception ट्राइ ब्लॉक के भीतर होता है, तो संबंधित कैच ब्लॉक execute होता है। विभिन्न प्रकार के exception को संभालने के लिए Multiple catch blocks का उपयोग किया जा सकता है।

3. finally:

finally block का उपयोग कोड के एक ब्लॉक को Specified करने के लिए किया जाता है जिसे exception होने या न होने की परवाह किए बिना execute किया जाएगा। इसका उपयोग आमतौर पर cleanup tasks or resources को जारी करने के लिए किया जाता है जिन्हें परिणाम की परवाह किए बिना execute करने की आवश्यकता होती है।

4. throw: 

किसी exception को स्पष्ट रूप से थ्रो करने के लिए थ्रो कीवर्ड का उपयोग किया जाता है। इसके बाद exception का उदाहरण या थ्रोएबल का Subclass आता है। इसका उपयोग आम तौर पर एक मेथड के भीतर यह संकेत देने के लिए किया जाता है कि कोई extraordinary situation उत्पन्न हो गई है।

5. throws: 

थ्रो कीवर्ड का उपयोग मेथड घोषणा में यह इंगित करने के लिए किया जाता है कि मेथड एक या अधिक प्रकार के exception थ्रो कर सकता है। यह उन exception प्रकारों को Specified करता है जिन्हें Method द्वारा थ्रो किया जा सकता है। Method का कॉलर उन exception को handle or promote करने के लिए ज़िम्मेदार है।

6. try-with-resources:

ट्राई-विथ-रिसोर्स स्टेटमेंट का उपयोग उन resources को automatically बंद करने के लिए किया जाता है जो ऑटोक्लोजेबल इंटरफ़ेस को लागू करते हैं। यह सुनिश्चित करता है कि resources ठीक से बंद हैं, भले ही कोई exception हो। resources को प्रयास statement के भीतर declared and initialized किया जाता है, और वे try block के अंत में automatically  बंद हो जाएंगे।
